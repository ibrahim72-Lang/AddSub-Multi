───────────────────────────────────────────────────────────────────────
  🔍 WHAT WAS WRONG
───────────────────────────────────────────────────────────────────────

The batch file (AddSub-Multi-6.0.bat) was designed for interactive use:
  • It prompted users for subfolder preferences
  • It prompted users for reprocess policy
  • It used `set /p` and `choice` commands for input

When called from the GUI:
  • GUI had no way to provide these inputs
  • Prompts failed silently (no console for user input)
  • Batch file used default behavior or exited early
  • GUI settings were stored but never used
  • No error messages were captured or displayed

───────────────────────────────────────────────────────────────────────
  🎯 HOW IT WORKS NOW
───────────────────────────────────────────────────────────────────────

STEP-BY-STEP FLOW:

1. User adds videos to GUI
2. User clicks "Start Processing"
3. GUI reads settings:
   • Include subfolders: checkbox state
   • Reprocess policy: radio button selection
4. GUI creates environment variables:
   • SUBFOLDER_CHOICE='Y' or 'N'
   • REPROCESS_POLICY='SKIP_ALL', 'REPROCESS_ALL', or 'ASK_EACH'
5. GUI calls batch file with these environment variables
6. Batch file checks if variables are already set:
   • If REPROCESS_POLICY is set → use it
   • If REPROCESS_POLICY is empty → prompt user (won't happen from GUI)
   • If SUBFOLDER_CHOICE is set → use it
   • If SUBFOLDER_CHOICE is empty → prompt user (won't happen from GUI)
7. Batch file processes videos normally
8. GUI captures all output:
   • Parses status for UI updates
   • Logs everything to console
   • Detects errors
9. When done:
   • If successful → "Complete" message
   • If failed → Error dialog with details
